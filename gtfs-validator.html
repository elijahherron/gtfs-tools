Validation Information

Static GTFS Checks

  • Required files present (agency, stops, routes, trips, stop_times)
  • Required fields in each file
  • Field format validation (lat/lon, times, dates, URLs)
  • Feed expiry dates (feed_info, calendar)
  • Referential integrity between files
  • Duplicate ID detection
  • Optional file presence

GTFS-rt Checks

  • Feed header (version, timestamp)
  • Entity structure validation
  • Required fields per entity type
  • Timestamp freshness
  • Trip descriptor completeness
  • Vehicle position accuracy

Combined Checks

  • trip_id matching
  • route_id matching
  • stop_id matching
  • Schedule relationship analysis
